ЛАБОРАТОРНА РОБОТА - Vue 3 + TypeScript + Vue Router

1. КРОКИ, ЯКІ Я ВИКОНАЛА:
- Створила проєкт Vue 3 командою npm create vue@latest vue-lab
- Обрала опції: TypeScript (Yes), Vue Router (Yes), ESLint (Yes), Prettier (Yes)
- Встановила залежності: npm install
- Запустила сервер: npm run dev
- Створила нову сторінку ContactView.vue у папці src/views
- Зареєструвала маршрут /contact у файлі src/router/index.ts
- Додала посилання Contact у навігацію в App.vue
- Створила компонент UserCard.vue з типізацією пропсів (username: string, isActive: boolean)
- Використала компонент UserCard на сторінці Contact
- Перевірила роботу TypeScript, передавши неправильний тип даних

2. ПОМИЛКА TYPESCRIPT (яку я отримала):
Коли я спробувала передати число замість boolean у компоненті UserCard:
<UserCard username="Test" :isActive="123" />

TypeScript одразу показав помилку:
"Type 'number' is not assignable to type 'boolean'"
VS Code підсвітив помилку червоним кольором ще до запуску програми. Це дуже зручно, бо можна виправити помилку відразу, не чекаючи, поки програма "зламається" під час роботи.

3. ВИСНОВКИ:
- TypeScript дуже корисний! Він ловить помилки на етапі написання коду, а не під час виконання програми
- Vue Router дозволяє створювати односторінкові застосунки (SPA) з навігацією без перезавантаження сторінки, що робить інтерфейс швидшим і зручнішим
- Структура проєкту Vue 3 логічна та зрозуміла: views для сторінок, components для багаторазових компонентів, router для маршрутів
- Типізація пропсів допомагає уникнути багатьох помилок і робить код надійнішим
- Робота з Vue 3 + TypeScript комфортна завдяки хорошій підтримці в VS Code
