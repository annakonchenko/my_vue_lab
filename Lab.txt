- Додала посилання Contact у навігацію в App.vue
- Створила компонент UserCard.vue з типізацією пропсів (username: string, isActive: boolean)

- Перевірила роботу TypeScript, передавши неправильний тип даних

2. ПОМИЛКА TYPESCRIPT (яку я отримала):
Коли я спробувала передати число замість boolean у компоненті UserCard:
<UserCard username="Test" :isActive="123" />

TypeScript одразу показав помилку:
"Type 'number' is not assignable to type 'boolean'"
VS Code підсвітив помилку червоним кольором ще до запуску програми. Це дуже зручно, бо можна виправити помилку відразу, не чекаючи, поки програма "зламається" під час роботи.

- Структура проєкту Vue 3 логічна та зрозуміла: views для сторінок, components для багаторазових компонентів, router для маршрутів

- Робота з Vue 3 + TypeScript комфортна завдяки хорошій підтримці в VS Code


ЛАБОРАТОРНА РОБОТА - Tailwind CSS

ЧАСТИНА 1: ВСТАНОВЛЕННЯ TAILWIND CSS
Для інсталяції Tailwind CSS у проєкт Vue 3 я виконала наступні кроки:

Крок 1: Встановлення пакетів

Спочатку я відкрила термінал у кореневій папці проєкту vue-lab і встановила необхідні пакети командою:
npm install -D tailwindcss @tailwindcss/vite
Ця команда встановила Tailwind CSS версії 4 та офіційний Vite plugin, який рекомендований для інтеграції з Vue 3.

Крок 2: Налаштування Vite
Потім я відкрила файл vite.config.js у кореневій папці проєкту і додала Tailwind plugin до конфігурації Vite. Я імпортувала @tailwindcss/vite та додала його до масиву plugins. Це дозволило Vite обробляти Tailwind CSS під час збірки проєкту.
Крок 3: Створення CSS файлу

Я створила новий файл src/assets/css/index.css і додала в нього єдиний імпорт:
@import "tailwindcss";

У Tailwind CSS версії 4 використовується саме такий спосіб підключення, на відміну від попередніх версій, де потрібно було писати три окремі директиви @tailwind.
Крок 4: Підключення стилів у додаток

Нарешті, я відкрила файл src/main.js і додала імпорт CSS файлу перед створенням Vue додатку:
import "./assets/css/index.css";
Це забезпечило глобальне підключення Tailwind стилів до всього проєкту.
Після виконання цих кроків я запустила сервер розробки командою npm run dev і переконалася, що Tailwind CSS успішно прцює у моєму проєкті. 
ЧАСТИНА 2: УМОВНІ СТИЛІ У USERCARD.VUE
Я реалізувала умовні стилі через директиву :class з тернарним оператором: :class="isActive ? 'border-green-500 bg-green-50' : 'border-gray-300 bg-white'"
Якщо isActive = true → зелена рамка і фон
Якщо isActive = false → сіра рамка і білий фон
Так само для тексту статусу:
:class="isActive ? 'text-green-700' : 'text-gray-500'"
Вся логіка стилізації в template, без окремого CSS коду.
ЧАСТИНА 3: ПЕРЕВАГИ ТА НЕДОЛІКИ TAILWIND
ПЕРЕВАГИ:
- Не потрібно писати CSS файли
- Швидка розробка
- Всі стилі в одному місці (в template)
- Менше коду загалом
НЕДОЛІКИ:
- Багато класів у HTML (виглядає перевантажено)
- Треба вчити назви класів Tailwind
- Довгі рядки коду в template
